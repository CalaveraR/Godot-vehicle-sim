# Core backend layout (Godot + Rust)

This repository now follows a per-domain split:

- `*/godot`: orchestration, engine integration, scene/node APIs.
- `*/rust`: pure math kernels, deterministic clamps, and test/golden generators.

Current status:
- Runtime orchestration still lives in existing domain roots (`engine/`, `tires/`, `wheels/`, etc.) to preserve scene compatibility.
- `tires/rust/` is the first active golden reference backend (`rust/tire_core` remains source crate).
- `TireRuntimeCoordinator` now exposes backend flags:
  - `core_backend = "gdscript" | "rust"`
  - `validate_parity = true/false`

Migration rule:
1. Keep Godot node/scene operations in GDScript.
2. Move pure math kernels to Rust first.
3. Validate GDScript parity with golden vectors generated by Rust.
